# 家庭积分管理系统 - 测试用例文档

## 1. 登录与安全
| ID | 功能模块 | 测试场景 | 预期结果 |
|---|---|---|---|
| TC-01 | 登录 | 使用正确的 PIN 码登录 | 成功进入系统，显示对应角色界面 |
| TC-02 | 登录 | 使用错误的 PIN 码登录 | 提示 PIN 码错误，无法进入 |
| TC-03 | 管理员 | 管理员 PIN 码验证 | 只有输入正确的管理员 PIN 才能进入管理后台 |
| TC-04 | 安全 | 环境变量检查 | 检查代码中是否还存在硬编码的 Supabase Key |

## 5. 奖励管理
| ID | 功能模块 | 测试场景 | 预期结果 |
|---|---|---|---|
| TC-16 | 奖励设置 | 管理员添加自定义奖励 | 奖励列表中出现新项，成员可查看 |
| TC-17 | 奖励状态 | 停用/启用奖励 | 停用的奖励在成员界面不可见 |
| TC-18 | 系统稳定性 | 代码重构后的编译检查 | 运行 `npm run build` 和 `npm run lint` 均通过，无语法错误或重复声明错误 |
| TC-19 | 存储优化 | 模拟 LocalStorage 满载 | 手动向 LocalStorage 写入大量数据后刷新页面，系统应能自动清理缓存并正常运行，不报 QuotaExceededError |
| TC-20 | 缓存优化 | 大尺寸头像/图片过滤 | 上传超过 20KB 的 base64 图片作为头像或评论图片，检查 LocalStorage 缓存中的对应项应为 null，但 UI 仍能通过 Supabase 正常显示 |
| TC-21 | 弹幕特效 | 多特效循环显示 | 发送一条选择了“闪烁”和“放大”两个特效的弹幕。观察弹幕第一次飞过屏幕时应为“闪烁”，第二次飞过时应为“放大”，第三次回到“闪烁”，以此类推。 |
| TC-22 | 自动化测试 | 运行单元测试 | 运行 `npx vitest run`，所有针对 `goalUtils` 和 `UserAvatar` 的测试用例应全部通过，确保核心逻辑未被破坏。 |
| TC-23 | 自动化测试 | 运行集成测试 | 运行 `npx vitest run`，模拟用户发送留言及任务卡片渲染的集成测试应全部通过。 |
| TC-24 | 系统监控 | 管理员健康检查 | 以管理员身份登录，页面底部应显示“系统自动化健康检查”面板，且各项指标（数据库、数据完整性、逻辑引擎）应显示为 PASS。 |
| TC-25 | 系统监控 | 手动全量测试 | 点击“开始执行全量测试”，系统应按“隔离性”、“功能”、“性能”三大维度依次运行 ISO/FUN/PER 系列测试项，并显示详细结果。 |
| TC-26 | 数据安全 | 隔离性验证 (ISO-01/02) | 运行测试时，拦截器应确保所有写操作不进入 Supabase，测试结束后内存状态应自动清理。 |
| TC-27 | 核心功能 | 任务生命周期 (FUN-01) | 模拟任务从创建到全员确认自动完成的完整路径，状态机切换应符合 PRD。 |
| TC-28 | 核心功能 | 积分奖励系统 (FUN-02) | 校验登录、留言、弹幕等多种渠道的积分触发逻辑，确保计分准确且无重复。 |
| TC-29 | 核心功能 | 成就解锁系统 (FUN-03) | 模拟满足成就条件，验证勋章是否自动解锁并同步发放额外积分。 |
| TC-30 | 核心功能 | 互动与聚合 (FUN-04) | 验证弹幕特效随机选择逻辑及留言板聚合显示功能。 |
| TC-31 | 核心功能 | 可视化引擎 (FUN-05) | 校验统计图表的数据处理逻辑，确保趋势图与成员细分数据一致。 |
| TC-32 | 核心功能 | 奖励兑换 (FUN-06) | 模拟个人及家庭里程碑奖励兑换，验证积分扣减与库存同步逻辑。 |
| TC-33 | 性能压力 | 基础压力测试 (PER-01/02) | 模拟 500 条数据渲染及并发冲突，系统应保持流畅且数据一致。 |
| TC-34 | 性能压力 | 数据库 I/O 时延 (PER-03) | 测量数据库响应时间，应显示具体的毫秒数，并根据时延给出性能评价。 |
| TC-35 | 性能压力 | 网络访问质量 (PER-04) | 探测当前网络环境到源站的延迟，应能识别网络质量（极佳/一般/较差）。 |
