# DDYY@ME 家庭积分管理系统 - 产品需求文档 (PRD)

## 1. 项目概述
一个旨在通过积分奖励机制促进家庭成员互动、任务完成和良好习惯养成的 Web 应用。品牌名称已升级为 **DDYY@ME**，采用现代艺术风格。

## 2. 核心功能

### 2.1 任务管理 (Task Center)
- **分类展示**: 
  - 第一层级：通过“与我相关”和“家庭全部”两个标签页进行切换。
  - 第二层级：提供“全部”、“待处理”和“已完成”三个过滤选项。
- **任务排序**: 
  - 优先显示“与我相关”且未完成的任务。
  - 其次显示“家庭相关”的任务。
  - 已完成的任务显示在最下方，且视觉上进行弱化处理（背景变暗、灰度处理）。
- **任务进度**:
  - 任务进度在所有责任人确认后自动达到 100%。
- **任务讨论**:
  - 支持在任务下进行评论和回复，支持 @ 提及功能。
  - **引用回复**: 回复他人留言时，会自动引用并显示原留言的简短内容。
  - **智能聚焦**: 点击回复时，输入框会自动获取焦点，光标自动定位到末尾，提升互动效率。
- **任务到期提醒**: 
  - 当任务时间消耗达到 80% 且未完成时，在页面顶部显示滚动提醒条。
  - 提醒条可手动关闭。
- **新建目标**: 
  - 支持快捷选择持续时间（1天、3天、1周、半个月、1个月）。
  - 默认持续时间为 1 周。

### 2.2 积分与奖励系统
- **积分获取**: 
  - 完成任务获得积分。
  - **每日登录奖励**: 每日首次打开网站自动获得 1 积分。
  - **活跃奖励**: 发送弹幕或在任务讨论中留言可获得 1 积分（每日上限 10 分）。
- **积分规则**: 
  - **基础分值**: 每个任务基础 10 分，由所有参与者平分（向上取整）。
  - **提前奖励**: 提前完成任务额外获得 3 分，由所有参与者平分（向上取整）。
  - **团队协作奖**: 如果是多人任务（参与人数 > 1），每位参与者额外获得 3 分。
  - **计算示例**: 
    - 2人完成非提前任务：(10/2) + 3 = 8 分/人。
    - 2人提前完成任务：ceil(13/2) + 3 = 10 分/人。
    - 1人完成提前任务：10 + 3 = 13 分/人。
- **奖励兑换**: 
  - 个人里程碑：达到特定积分后解锁个人奖励。
  - 家庭里程碑：全家总积分达到特定值后解锁家庭奖励。

### 2.3 弹幕留言板 (Danmaku Board)
- **实时弹幕**: 支持发送带特效（颜色、大小、速度、动画效果）的弹幕。
- **配置面板**: 点击调色盘图标展开配置面板，包含：
  - **快捷表情**: 常用表情快捷栏，支持一键发送。
  - **样式配置**: 颜色、字体大小、滚动速度、动画特效选择。
- **滚动稳定性**: 弹幕动画采用 `100vw` 到 `-100%` 的完整位移逻辑，确保任何长度的弹幕都能完整穿过屏幕并正常循环，彻底解决卡死问题。
- **第一名标识**: 积分排名第一的成员发布的弹幕会带有皇冠图标和金色边框。
- **弹幕开关**: 使用“眼睛”图标控制弹幕显示/隐藏。
- **排行可见性**: 优化排行图表比例，确保所有成员（包括 0 分成员）的头像和积分清晰可见。

### 2.4 最近动态与积分增长 (Activity & Growth)
- **动态追踪**: 自动记录家庭成员的关键行为（登录、弹幕、任务创建/完成）。
- **展示方式**: 
  - 位于弹幕留言板下方，采用并列布局。
  - 动态按时间倒序排列，支持实时更新。
  - **滚动条优化**: 动态面板高度固定，内容过多时自动出现滚动条，支持平滑滚动查看所有历史动态。

### 2.5 视觉设计
- **品牌标识**: 顶部展示 **DDYY@ME**，采用黑体斜体 (Black Italic) 与橙粉渐变艺术效果，并保留版本号标签。
- **实时反馈**: 留言板显示“实时同步中”状态标签。

### 2.6 数据可视化 (Data Visualization)
- **家庭总积分增长曲线**: 
  - 在页面顶部 Hero 区域展示家庭总积分的最近 7 天增长趋势。
  - **日期轴**: 横轴增加日期显示。

### 2.7 成员管理
- **PIN 码登录**: 每个成员有独立的 4 位 PIN 码。
- **管理员权限**: 管理员可以修改所有成员的 PIN 码和管理数据。

## 3. 技术实现
- **前端**: React, Tailwind CSS, Framer Motion, Lucide React.
- **后端/数据库**: Supabase (PostgreSQL, Real-time).
- **状态管理**: React Hooks, SWR.
