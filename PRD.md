# 家庭积分管理系统 (Family Points System) - 产品需求文档 (PRD)

## 1. 项目概述
一个旨在通过积分奖励机制促进家庭成员互动、任务完成和良好习惯养成的 Web 应用。

## 2. 核心功能

### 2.1 任务管理 (Task Center)
- **分类展示**: 
  - 第一层级：通过“与我相关”和“家庭全部”两个标签页进行切换。
  - 第二层级：提供“全部”、“待处理”和“已完成”三个过滤选项。
- **任务排序**: 
  - 优先显示“与我相关”且未完成的任务。
  - 其次显示“家庭相关”的任务。
  - 已完成的任务显示在最下方，且视觉上进行弱化处理（灰度、低透明度）。
- **任务到期提醒**: 
  - 当任务时间消耗达到 80% 且未完成时，在页面顶部显示滚动提醒条。
  - 提醒条可手动关闭。
- **新建目标**: 
  - 支持快捷选择持续时间（1天、3天、1周、半个月、1个月）。
  - 默认持续时间为 1 周。

### 2.2 积分与奖励系统
- **积分获取**: 
  - 完成任务获得积分。
  - **每日登录奖励**: 每日首次打开网站自动获得 1 积分（无需手动执行任务）。
- **奖励兑换**: 
  - 个人里程碑：达到特定积分后解锁个人奖励。
  - 家庭里程碑：全家总积分达到特定值后解锁家庭奖励。
- **积分规则**: 
  - 积分仅用于追踪进度，不会被扣除（里程碑模式）。

### 2.3 弹幕留言板 (Danmaku Board)
- **实时弹幕**: 支持发送带特效（颜色、大小、速度、动画效果）的弹幕。
- **第一名标识**: 积分排名第一的成员发布的弹幕会带有皇冠图标和金色边框，以示荣誉。
- **留言板排序**: 留言按时间倒序排列（最新在前），默认显示 3 条，其余折叠。

### 2.4 最近动态 (Recent Activity)
- **动态追踪**: 自动记录家庭成员的关键行为，包括：
  - **登录**: 成员进入系统的时间。
  - **发布弹幕**: 成员发送留言的时间及简要内容。
  - **创建任务**: 成员新建目标的时间及目标名称。
  - **完成任务**: 成员达成目标的时间及目标名称。
- **展示方式**: 
  - 位于弹幕留言板下方。
  - 默认处于收起状态，仅展示最新的一条动态。
  - 点击可展开查看最近 50 条历史动态。
  - 动态按时间倒序排列，支持实时更新。

### 2.5 成员管理
- **PIN 码登录**: 每个成员有独立的 4 位 PIN 码。
- **管理员权限**: 管理员可以修改所有成员的 PIN 码和管理数据。

## 3. 技术实现
- **前端**: React, Tailwind CSS, Framer Motion, Lucide React.
- **后端/数据库**: Supabase (PostgreSQL, Real-time).
- **状态管理**: React Hooks, SWR.
