# DDYY@ME - 发布测试用例集 (v1.2.0)

为了确保系统的稳定性和功能的完整性，在每次发布新版本前，请务必执行以下测试用例。

## 1. 核心功能测试

### 1.1 目标管理
- [ ] **新建目标**：点击“新建目标”，填写名称、描述、责任人、优先级，选择“个人”或“家庭”类型，保存后应在列表中显示。
- [ ] **目标可见性**：确认“个人目标”仅在责任人的“与我相关”中显示，而“家庭目标”在“家庭全部”中对所有人可见。
- [ ] **编辑目标**：点击目标卡片右上角的编辑图标，修改内容并保存，内容应实时更新。
- [ ] **进度更新**：点击“进度+10%”，进度条应增长，且数据库同步成功。
- [ ] **回复聚焦**：在任务讨论区点击“回复”，输入框应自动获取焦点，光标位于末尾。

### 1.2 弹幕系统 (新增强化)
- [ ] **配置面板**：点击调色盘图标，应能看到快捷表情栏和各项配置参数。
- [ ] **快捷表情**：在配置面板中点击表情按钮，弹幕应立即发送且表情正确。
- [ ] **滚动稳定性**：发送超长弹幕，观察其是否能完整穿过屏幕并正常消失/循环，无卡死现象。
- [ ] **开关状态**：点击“眼睛”图标关闭弹幕，刷新页面后弹幕应保持关闭状态。

### 1.3 确认机制 (核心逻辑)
- [ ] **标记完成**：点击“确认完成”按钮，进度应变为 99%，卡片状态变为“待确认”。
- [ ] **全员确认**：切换不同角色，分别点击“确认完成”按钮。
- [ ] **自动结算**：当最后一名成员确认后，进度应自动变为 100%，状态变为“已完成”。
- [ ] **积分计算**：验证单人与多人任务的积分分配逻辑是否符合 PRD 规则（基础分平分 + 团队奖每人3分）。
- [ ] **积分拆分**：确认交易记录中显示了积分拆分详情（基础、提前、团队）。

### 1.4 界面与视觉
- [ ] **品牌展示**：顶部标题应显示为渐变色的 **DDYY@ME**，且带有 `V2.4全自动化版` 标签。
- [ ] **状态检查**：确认页面上显示 `实时同步中` 标签。
- [ ] **动态功能**：确认“最近动态”列表支持平滑滚动，且无“查看更多”按钮。

## 2. 权限与安全测试

### 2.1 角色切换
- [ ] **退出登录**：点击“退出”，应返回角色选择界面。
- [ ] **管理员登录**：输入正确的账号密码（admin/password），应成功进入管理界面。
- [ ] **操作限制**：普通成员不能删除他人创建的目标（除非是管理员）。

## 3. 数据可靠性测试

### 3.1 实时同步
- [ ] **多端同步**：在一个窗口修改数据，另一个窗口（如果打开）应能通过 Supabase Realtime 实时看到变化。
- [ ] **刷新恢复**：刷新页面后，所有数据（目标、积分、排行榜）应能正确从数据库加载。

### 3.2 备份与恢复
- [ ] **导出数据**：点击“数据管理” -> “导出备份”，应下载一个包含所有数据的 JSON 文件。
- [ ] **导入数据**：点击“导入备份”，选择文件后，系统数据应更新为备份文件中的内容。

---
**维护者提示**：每次修改 `App.tsx` 中的核心逻辑后，请至少抽检 1.2 和 1.3 项。
