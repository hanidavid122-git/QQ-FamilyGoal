# 多盈家庭目标 - 发布测试用例集 (v1.1.1)

为了确保系统的稳定性和功能的完整性，在每次发布新版本前，请务必执行以下测试用例。

## 1. 核心功能测试

### 1.1 目标管理
- [ ] **新建目标**：点击“新建目标”，填写名称、描述、责任人、优先级，保存后应在列表中显示。
- [ ] **编辑目标**：点击目标卡片右上角的编辑图标，修改内容并保存，内容应实时更新。
- [ ] **删除目标**：点击删除图标，确认后目标应从列表中消失。
- [ ] **进度更新**：点击“进度+10%”，进度条应增长，且数据库同步成功。

### 1.2 确认机制 (核心逻辑)
- [ ] **标记完成**：点击“确认完成”按钮，进度应变为 99%，卡片状态变为“待确认”。
- [ ] **全员确认**：切换不同角色（爸爸、妈妈、姐姐、妹妹），分别点击“确认完成”按钮。
- [ ] **自动结算**：当最后一名成员确认后，进度应自动变为 100%，状态变为“已完成”。
- [ ] **积分计算 (新规则)**：
    - 单人任务：获得 10 积分（若提前则 +3）。
    - 多人任务：(10 + 提前3 + 团队5) / 人数，向上取整。验证不同人数下的分配是否正确。
- [ ] **动态显示**：在“最近动态”中查看，应显示具体的活动内容及获得的积分数（如：`进入了系统 (+1 积分)`）。

### 1.3 积分与奖励
- [ ] **积分计算**：完成任务后，个人积分和家庭总积分应正确增加。
- [ ] **奖励兑换**：选择一个积分足够的成员，点击“兑换”奖励，积分应扣除，且生成一条“redeemed”类型的交易记录。
- [ ] **管理奖励**：以“管理员”身份登录，应能看到“管理奖励”按钮；以普通角色登录，该按钮应隐藏。

## 2. 权限与安全测试

### 2.1 角色切换
- [ ] **退出登录**：点击“退出”，应返回角色选择界面。
- [ ] **管理员登录**：输入正确的账号密码（admin/password），应成功进入管理界面。
- [ ] **操作限制**：普通成员不能删除他人创建的目标（除非是管理员）。

## 3. 数据可靠性测试

### 3.1 实时同步
- [ ] **多端同步**：在一个窗口修改数据，另一个窗口（如果打开）应能通过 Supabase Realtime 实时看到变化。
- [ ] **刷新恢复**：刷新页面后，所有数据（目标、积分、排行榜）应能正确从数据库加载。

### 3.2 备份与恢复
- [ ] **导出数据**：点击“数据管理” -> “导出备份”，应下载一个包含所有数据的 JSON 文件。
- [ ] **导入数据**：点击“导入备份”，选择文件后，系统数据应更新为备份文件中的内容。

---
**维护者提示**：每次修改 `App.tsx` 中的核心逻辑后，请至少抽检 1.2 和 1.3 项。
